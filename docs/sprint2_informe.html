3 Fichas de Caso de Uso

CU01 Registrar Jugador Actor: Usuario no registrado
Precondición
El email no debe estar registrado previamente en el sistema.
Flujo principal
  1. El usuario accede al formulario de registro público.
  2. Completa: nombre, apellido, email, teléfono, contraseña (dos veces) y categoría.
  3. El sistema valida que ambas contraseñas coincidan.
  4. El sistema valida que se haya seleccionado una categoría.
  5. El sistema aplica hash bcrypt(10) a la contraseña.
  6. El sistema inserta el registro en jugador con rol='jugador'.
  7. El sistema devuelve los datos del jugador creado (sin contraseña).
Flujos alternativos
Email duplicado → error 400. Contraseñas distintas → error 400. Sin categoría → error 400.
Postcondición
Nuevo registro en tabla jugador. El usuario puede iniciar sesión.
Notas
El registro de organizadores usa un endpoint separado (POST /registro-organizadores) protegido por ADMIN_SECRET. No está disponible en la interfaz pública.
CU02 Iniciar Sesión Actor: Jugador u Organizador
Precondición
El usuario debe tener cuenta registrada en el sistema.
Flujo principal
  1. El usuario ingresa email y contraseña.
  2. El sistema busca al jugador por email (búsqueda insensible a mayúsculas).
  3. El sistema valida la contraseña con bcrypt.compare().
  4. El sistema registra el intento en audit_log_ingresos (exitoso = true).
  5. El sistema genera y devuelve un token JWT con expiración de 6 horas.
  6. El frontend almacena el token y datos del usuario en localStorage.
Flujos alternativos
Email no encontrado → error 404, log con exitoso=false. Contraseña incorrecta → error 401, log con motivo.
Postcondición
Token JWT activo (6h). Acceso habilitado a recursos protegidos según rol.

CU03 Crear Torneo Actor: Organizador
Precondición
Usuario autenticado con rol='organizador'.
Flujo principal
  1. El organizador ingresa nombre, fechas (inicio, fin, cierre) y cupo máximo de equipos.
  2. Selecciona el formato de categoría: "Categoría Fija" o "Suma".
  3. Si Categoría Fija: selecciona una categoría (2da a 8va). Si Suma: ingresa el número objetivo (ej: 9 = 5ta+4ta).
  4. Selecciona la modalidad: "Fin de semana" o "Liga".
  5. Si Liga: ingresa los días de juego (ej: "miercoles,viernes").
  6. El sistema valida la combinación formato + categoría/suma.
  7. El sistema inserta el nuevo torneo en la tabla torneo.
Restricciones
categoria_fija requiere categoria_id. suma requiere suma_categoria numérico. Ambos son mutuamente excluyentes.
Postcondición
Nuevo torneo disponible en estado "Inscripción abierta". Los jugadores pueden inscribirse hasta la fecha de cierre.
CU04 Inscribir Equipo Actor: Organizador
Precondición
Torneo en período de inscripción. Ambos jugadores registrados en el sistema.
Flujo principal
  1. El organizador selecciona el torneo y los dos jugadores de la pareja.
  2. El sistema verifica que la inscripción esté dentro del período válido.
  3. El sistema verifica que ningún jugador esté ya inscripto en este torneo.
  4. El sistema verifica que no haya superposición de fechas con otros torneos.
  5. El sistema valida categorías según el formato del torneo:
    • categoria_fija: ambos jugadores deben tener la misma categoría que el torneo.
    • suma: la suma de valor_numerico de ambos debe igualar suma_categoria.
  6. El sistema crea un equipo con nombre auto-generado "Apellido1/Apellido2".
  7. El sistema registra la inscripcion.
Flujos alternativos
Categorías incompatibles → error 400 con detalle. Superposición de fechas → error 400. Cupo completo → error 400.
Postcondición
Nuevo registro en equipo e inscripcion. El equipo aparece en la lista del torneo.

CU05 Generar Grupos y Fixture Actor: Organizador / Sistema
Precondición
Torneo con al menos 2 equipos inscriptos. Grupos no generados previamente.
Flujo — Fin de Semana
  1. El sistema asigna los equipos aleatoriamente a grupos de 3-5 equipos.
  2. Por cada grupo genera partidos round-robin (todos contra todos).
  3. Crea registros en grupos, equipos_grupo y partidos_grupo.
Flujo — Liga
  1. El sistema crea un único grupo llamado "Liga Única" con todos los equipos.
  2. Genera un fixture round-robin completo (todos contra todos).
  3. Asigna fechas a cada jornada usando los dias_juego configurados en el torneo.
  4. Actualiza automáticamente la fecha_fin del torneo con la fecha de la última jornada.
Postcondición
Partidos de grupos disponibles en el sistema. En modo liga, cada partido tiene fecha asignada.
CU06 Cargar Resultado de Partido Actor: Organizador
Precondición
Torneo con grupos generados. Partido en estado no_iniciado.
Flujo — Fase de Grupos
  1. El organizador selecciona el torneo y la pestaña Grupos.
  2. Puede filtrar partidos por nombre de equipo o jugador (barra de búsqueda).
  3. En modo liga: puede filtrar por jornada usando el desplegable de fechas.
  4. Selecciona un partido y hace clic en "Cargar Resultado".
  5. Ingresa el score de cada set: games del equipo 1 vs games del equipo 2.
  6. El sistema calcula sets ganados y determina el ganador (no es manual).
  7. Actualiza partidos_grupo (estado = 'finalizado') y las estadísticas en equipos_grupo.
  8. Si todos los partidos del torneo quedan finalizados → genera el playoff automáticamente.
Flujo — Play-off
  1. El organizador selecciona la pestaña Play-off.
  2. Visualiza el bracket organizado por rondas (CUARTOS → SEMIS → FINAL).
  3. Carga el resultado del cruce (mismo mecanismo: sets por score).
  4. El sistema calcula el ganador y lo pasa automáticamente al siguiente cruce.
  5. Al completar la FINAL, el sistema genera los registros de ranking del torneo.
Nota importante
El sistema no usa un campo "ganador" seleccionado manualmente. El ganador se determina siempre contando sets ganados a partir del score ingresado.
Postcondición
Resultado guardado, estadísticas actualizadas, ganador avanza en el bracket.

CU07 Generar Play-off Actor: Sistema (automático) u Organizador (manual)
Precondición
Todos los partidos de la fase de grupos finalizados. Playoff no generado previamente.
Lógica — 1 Grupo (Liga)
Bracket dinámico según cantidad de equipos:
  • ≥ 8 equipos: los 8 primeros → CUARTOS + SEMIS + FINAL
  • ≥ 4 equipos: los 4 primeros → SEMIS + FINAL
  • < 4 equipos: los 2 primeros → FINAL directa
Lógica — 2 Grupos
Top 2 de cada grupo (4 equipos) → SEMIS + FINAL.
Lógica — 3 Grupos
Top 2 de cada grupo (6 equipos) + los 2 mejores terceros ("mejor tercero") = 8 equipos → CUARTOS + SEMIS + FINAL.
Lógica — 4+ Grupos
Top 2 de cada grupo → CUARTOS u OCTAVOS según total de clasificados (snake-seeding).
Seeding (bracket)
Formato estándar: 1° vs 8°, 4° vs 5°, 2° vs 7°, 3° vs 6°. Cada partido tiene next_match_id y next_slot para el avance automático del ganador.
Postcondición
Registros en partidos_llave con vínculos de avance automático configurados.
CU08 Consultar Ranking Actor: Jugador (público, sin login)
Precondición
Al menos un torneo completado con resultado final registrado.
Flujo principal
  1. El usuario accede a la página de Ranking (no requiere autenticación).
  2. Puede filtrar por categoría usando el desplegable.
  3. El sistema consulta ranking_jugador ordenado por puntos DESC.
  4. Se muestra: posición, jugador, última pareja, último torneo, fase alcanzada y puntos totales.
Escala de puntos
Campeón: 2000 · Subcampeón: 1200 · Semis: 720 · Cuartos: 360 · Octavos: 180
Postcondición
Solo lectura — no modifica datos.
